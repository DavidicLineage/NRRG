# Changelog

All notable changes to this project will be documented in this file.

This repository is currently in seed-theory stage. Versions reflect conceptual development and public release discipline, not claims of finality.

## [0.1.0] - 2026-03-09

### Added
- Initial repository structure for the **Nested Recursive/Recombinant Generativity** framework
- `README.md` introducing the framework, scope, limits, and safety posture
- `definitions.md` establishing core terminology:
  - recursion
  - recombinance
  - recombination
  - nested generativity
  - inheritance
  - distinct source
  - constraint
  - selection
  - invariant
  - major failure-mode terms
- `domain-mappings.md` mapping the framework across:
  - genetics
  - language
  - culture
  - cognition
  - artificial reasoning
  - temporality/spatiality (provisional)
- `failure-modes.md` identifying major productive-path failure risks, including:
  - recursive lock
  - recombinant noise
  - median freeze
  - extreme shatter
  - attractor tyranny
  - false fertility
  - memetic capture
  - constraint erosion
  - runaway adaptive sludge
  - premature grand unification
  - false ontologization
  - misaligned design acceleration
- `seed-theory.md` providing the first consolidated formulation of the theory
- `open-questions.md` documenting major unresolved problems, attack surfaces, and research directions

### Established
- Initial distinction between:
  - **recursion** as generativity through self-return
  - **recombinance** as generativity through composition of distinct sources
- Central nested claim:
  - recursively matured structures become substrates for recombinant interaction
  - recombinant products become substrates for further recursive inheritance or refinement
- Genetics as the canonical natural example of nested recursive/recombinant generativity
- Cross-domain hypothesis that related architecture may appear in language, culture, cognition, and artificial reasoning
- Early safety posture treating constraint architecture as primary for any future design application

### Notes
- This is the first public seed release of the framework
- The theory remains explicitly provisional
- Temporality/spatiality mapping remains speculative
- No formal mathematical treatment has yet been added
- No code is currently included in the repository
